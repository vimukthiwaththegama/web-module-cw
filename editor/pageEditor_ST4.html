Profile Page

Content Overview

The Profile Page for Wellness Hub showcases the details of Sarah Johnson â€“ a Health Advocate & Wellness Specialist. It features a header card with a profile avatar (with hover effects), detailed profile information (name and professional role), and multiple sections including About Me, Skills, Goals, Volunteer Experience, UN Goals, Availability, and a Contact form.

Technical Description

Built with semantic HTML elements such as <header>, <nav>, <section>, and <footer>, the page uses CSS Flexbox for responsive layout and CSS transitions for hover effects. <nav> contains <ul> and <li> elements for structured navigation. The CSS Flexbox layout allows responsive design, adapting to different screen sizes seamlessly.<table> is used to display availability in a clear, organized format, while <form> ensures a clean, modern form layout. CSS transitions are applied for smooth hover effects for Profile Avatar ,Navigation Links,Goal Cards & Volunteer Experience,Contact Form Button, UN Goals Section Image.

Accessibility

The page includes descriptive alt text for images, uses ARIA roles (e.g., role="img" with aria-label on the avatar), and follows a logical, semantic HTML structure to support screen readers. The contact form is accessible with properly labeled fields and required attributes.

Site Map Page

Content Overview

The Site Map Page organizes all pages of the Wellness Hub website into a clean, clickable list, providing easy navigation to Home, Profile, Volunteer, Table, Feedback, Team, and other pages.

Technical Description

Constructed with a semantic <section> and an unordered list (<ul>), the page utilizes simple CSS for spacing, hover effects on links, and responsive design principles.

Accessibility

By using semantic list elements and descriptive link texts, the Site Map Page is easily navigable for users with assistive technologies.

Challenges and Lessons Learned

Developing the Wellness Hub site presented several challenges:

  • Responsive Design: Iterative testing and media queries were necessary to ensure a consistent experience across devices.
  • Consistent Styling: Balancing unique content elements with a cohesive design demanded careful planning and testing.
  • Accessibility: Implementing semantic HTML and ARIA attributes across pages required meticulous attention to detail.

Compliance

All pages adhere to W3C HTML5 and CSS3 standards, incorporate WCAG 2.1 accessibility guidelines, and follow security best practices such as using rel="noopener" on external links.
